# 简单待办软件

一个使用HTML/CSS/JavaScript实现的简单待办软件，用于记录每天的任务。

## 功能特点

- ✅ **添加任务**：输入任务内容并添加到列表
- ✅ **完成任务**：勾选复选框标记任务为完成状态
- ✅ **编辑任务**：点击编辑按钮修改任务内容
- ✅ **删除任务**：点击删除按钮移除任务
- ✅ **本地存储**：数据保存在浏览器本地存储中，刷新页面不丢失
- ✅ **统计信息**：显示总任务数、已完成数和待完成数
- ✅ **响应式设计**：适配电脑、平板和手机屏幕
- ✅ **日期显示**：显示当前日期
- ✅ **示例任务**：第一次使用时可选添加示例任务

## 使用方法

1. 双击 `index.html` 文件在浏览器中打开
2. 在输入框中输入任务内容
3. 点击"添加任务"按钮或按Enter键添加任务
4. 勾选任务前的复选框标记为完成
5. 使用编辑按钮修改任务内容
6. 使用删除按钮移除任务

## 文件结构

## 技术细节

- **纯前端实现**：无需服务器，直接在浏览器中运行
- **本地存储**：使用`localStorage`保存任务数据
- **响应式设计**：使用CSS媒体查询适配不同屏幕尺寸
- **无外部依赖**：仅使用Font Awesome图标库（CDN）
- **跨浏览器兼容**：支持现代浏览器（Chrome、Firefox、Edge、Safari）

## 自定义修改

你可以直接编辑`index.html`文件来自定义应用：

1. **修改样式**：在`<style>`标签中修改CSS
2. **修改功能**：在`<script>`标签中修改JavaScript
3. **修改文字**：直接修改HTML中的中文文本

## 注意事项

- 数据保存在当前浏览器的本地存储中，清除浏览器数据会删除所有任务
- 建议定期备份重要任务
- 每个任务最多支持100个字符

## 日志导出

项目包含一个日志导出脚本，用于将localStorage中的工作日志导出为Markdown文件。

### 安装和使用

```bash
# 安装依赖
npm install

# 导出日志
npm run export:logs
```

脚本会：
1. 启动本地服务器
2. 智能检测localStorage数据源：优先`work_logs`，然后`dailyTasks`
3. 格式化数据：`dailyTasks`转换为Markdown任务列表，`work_logs`保持原样
4. 根据数据源保存文件：`todos-YYYY-MM-DD.md`、`logs-YYYY-MM-DD.md`或`empty-YYYY-MM-DD.md`

### 要求
- Node.js 18.0.0 或更高版本
- 自动安装Playwright浏览器（首次运行可能需要时间）

### 故障排除
- 智能检测：脚本优先查找`work_logs`，然后`dailyTasks`，如果都不存在会显示所有可用键
- `dailyTasks`数据格式：应为JSON数组，包含`{id, text, completed, createdAt}`字段
- 确保在项目根目录运行命令
- 检查localStorage：在浏览器开发者工具中确认有数据
- Windows用户：脚本已测试兼容

### 输出示例

当导出 `dailyTasks` 数据时，会生成如下格式的Markdown文件：

```markdown
# 待办事项 - 2026-03-03

## 未完成 (2)

- [ ] 完成项目文档 (创建时间: 2026/3/3 下午2:30:00)
- [ ] 测试导出功能 (创建时间: 2026/3/3 下午3:15:00)

## 已完成 (1)

- [x] 安装项目依赖 (创建时间: 2026/3/3 上午10:00:00)

## 统计
- 总任务数: 3
- 已完成: 1
- 未完成: 2
```

`work_logs` 数据会保持原样导出，不做格式化处理。
